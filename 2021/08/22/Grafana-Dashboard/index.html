Grafana Dashboard

How to represent data from IoT gateway on Grafana dashboard

Outline:

  1. Purpose
  2. Architecture and prerequisite
  3. Install and configure a mosquitto MQTT broker
  4. Install and configure Node-red
  5. Install and configure InfluxDB
  6. Install and configure Grafana Dashboard
  7. Summary

1. Purpose

This post aims to help blog reader to quickly build up a demonstration to see mqtt data from Grafana Dashboard without coding

2. Architecture and prerequisite

In the demonstration, we will use Moxa UC-8200 series ARM-based embedded computer to generate system infomration (cpu/memory usage) and stream it to the MQTT Broker. This IoT gateway has the ability to convert OT data like Modbus TCP/RTU to MQTT/Azure/AWS. In this tutorial, we will only use the gateway system information to simplify this demonstration.

We will use this IoT gateway to send MQTT message to a MQTT broker. To do so, we need to install the Mosquitto MQTT broker.

Grafana dashboard doesn’t come with native MQTT streaming data source. As the result, I plan to use influxDB as the data source.

Node-red can ingest the MQTT message and write those data into influxDB. The benefits of using node-red is we don’t need to code anything, and we can just use the existing component to write MQTT data in json format of data to influxDB

All the service I mentioned above can be installed locally in any private network on a Linux server, or you can install all the services on the virtual machine on the public cloud. During my demonstration, I will just deploy all the services on my virtual machine on Azure.

You will need to set up a firewall rule to ensure all those service port be allowed to connect from outside.

Services Used TCP Port
Mosquitto MQTT Broker 1883
Node-red 1880
InfluxDB 8086
Grafana Dashboard 3000

3. Install and configure a mosquitto MQTT broker

In this step, we will need to install mosquitto MQTT broker.

Check the mqtt broker run successfully

Once the MQTT broker run successfully, we can go to setup the MQTT message source.
While using Moxa’s IoT gateway, we need to fill in the broker IP address, which should be your linux server IP address, and port 1883
SSL/TLS is not enabled in this demonstration.

The uploaded message content should follow a flat data schema in a json object format - property: value

We can use tool like MQTT.fx to subscribe to the topic which published by the MQTT message source.

4. Install and configure Node-red

Before we start to install node-red, we need to install nodejs and npm

Then, we can start to install node-red.
To install it on the Ubuntu server hosted by Microsoft Azure
We follow the [step-by-step instruction for azure] (https://nodered.org/docs/getting-started/azure)

If you want it to run automatically after system booting up, you can follow the step below

After you finish all the step above, you can login to the node-red web ui by the server-ip:3000

Before we deep dive into the node introduction, let us have an overview of what we are going to do with Node-red
Starting from the left hand side, we need to have a MQTT client node to subscribe the data from MQTT broker.
The incoming message can be checked by using debug node, and you can see the data type of the message is ‘string’
We need to have a json component to convert the string data to a json object data.
In the right hand side, influxDB component save the incoming data from json node and save it to the InfluxDB

Let’s start from the ‘mqtt in’ node

#Step1. Drag the ‘mqtt in’ node from the left node menu to the workplace and configure it.
Double click the node and start to edit the node configuration.
We will need to fill in the MQTT broker IP address, listen port, and other required setting

#Step2. Drag the ‘debug’ node from the left node menu to the workplace
link ‘mqtt in’ and ‘debug’ node together to pipe all the received MQTT message to the debug node.
click the debug message from the right menu and you can see all the messages which is in datatype string

#Step3. Drag the ‘json’ node from the left menu to the workpalce
link ‘mqtt in’ node and ‘json’ node together, so json node can convert all the incoming string type data to json format data
From the figure below, you can see all the datatype turn to json object from the debug message

#Step4. Install the node-red-contrib-influxdb from Manage palette
Go to Manage palette and search for node-red-contrib-influxdb and install it.

#Step5. Drag the ‘influxDB out’ node from the left menu to the workplace and configure it
link ‘json’ node and ‘influxDB out’ together, so influxDB node can save all the json data into database.
Double click the node and fill in the required configuration like bucket name, organization, server version
, and mearsurement. A token is required to successfuly connect to the influxDB.

Since we haven’t configured influxDB yet, we can keep the setting blank and go back to setup the setting once we finish the influxDB setup

Install the latest version 2.0 - https://docs.influxdata.com/influxdb/v2.0/install

You should see the influxdb service status running
Then, login to the linux console and start to setup the influxDB database
