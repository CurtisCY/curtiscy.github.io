Grafana Dashboard

How to represent data from IoT gateway on Grafana dashboard

Outline:

  1. Purpose
  2. Architecture and prerequisite
  3. Install and configure a mosquitto MQTT broker
  4. Install and configure Node-red
  5. Install and configure InfluxDB
  6. Install and configure Grafana Dashboard
  7. Summary

1. Purpose

This post aims to help blog reader to quickly build up a demonstration to see mqtt data from Grafana Dashboard

2. Architecture and prerequisite

In the demonstration, I have used Moxa UC-8200 series ARM-based embedded computer to generate system infomration (cpu/memory usage) and streamm it to the MQTT Broker. This IoT gateway has the ability to convert OT data like Modbus TCP/RTU to MQTT/Azure/AWS. However, in this tutorial, I just used the gateway system information to simplify the demonstration.

We will use this IoT gateway to send MQTT message to a MQTT broker. To do so, weneed to install the Mosquitto MQTT broker.

Grafana dashboard doesn’t come with native MQTT streaming data source. As the result, I plan to use influxDB as the data source. Now, the problem is how I can convert the data from MQTT to InfluxDB without coding.

One of the solution I found is to use Node-red to ingest the MQTT message and write those data into influxDB. The benefits of using node-red is we don’t need to code anything, and we can just use the existing component to achieve the goal.

All the service I mentioned above can be installed locally in any private network on a Linux server, or you can install all the services on the virtual machine on the public cloud. During my demonstration, I will just deploy all the services on my virtual machine on Azure.

You will need to set up a firewall rule to ensure all those service port be allowed to connect from outside.

Services Used TCP Port
Mosquitto MQTT Broker 1883
Node-red 1880
InfluxDB 8086
Grafana Dashboard 3000
